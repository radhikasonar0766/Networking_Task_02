#Networking_Task_02


**Prepared By:** Radhika Sonar

## Overview

This project explains how network communication works when a user opens **[www.google.com](http://www.google.com)** in a web browser. It demonstrates the journey of a request from a user's device to Google's server and the response returned back to the device.

## Communication Flow

1. **Your Device**

   * The user enters `www.google.com` in a web browser.
   * The device creates a request to access the website.

2. **Router**

   * The request is sent to the router.
   * The router forwards the request to the internet and acts as the gateway between the local network and external networks.

3. **DNS Server**

   * The Domain Name System (DNS) server receives a request to resolve the domain name.
   * DNS converts `www.google.com` into its corresponding IP address.

4. **Google Server**

   * Using the IP address obtained from DNS, the request reaches Google's web server.
   * The server processes the request and prepares the webpage data.

5. **Response Back to Device**

   * Google's server sends the webpage data back through the internet.
   * The router forwards the response to the user's device.
   * The browser renders the webpage and displays it to the user.

## Simple Flow Diagram

Your Device → Router → DNS Server → Google Server → Response Back to Device

## Importance of DNS

DNS is important because computers communicate using IP addresses, while humans use domain names. DNS translates domain names into IP addresses, allowing devices to locate and communicate with the correct servers.

## Conclusion

The network communication process involves multiple components working together. DNS resolves domain names, routers forward traffic, servers process requests, and responses are returned to the user, enabling access to websites such as Google.
